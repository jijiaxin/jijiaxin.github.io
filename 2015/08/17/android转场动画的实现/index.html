Android转场动画的实现

2015-08-17 23:14:54

要想提高android用户体验，转场动画是必不可少的一个耍酷技能
android转场动画可分为windowAnimation和ActivityAnimation，见名知意，一个是window级，一个是Activity级别

WindowAnimation 与 ActivityAnimation 比较

主要区别有：

  • WindowAnimation包括 windowEnterAnimation 和 windowExitAnimation ；
    ActivityAnimation包括 activityOpenEnterAnimation， activityOpenExitAnimation ， activityCloseEnterAnimation 和 activityCloseExitAnimation
  • 在项目中WindowAnimation优先级大于Activity，即在Activity转场过程中，如果同时设置了WindowAnimation和ActivityAnimation,那么很可能只会执行WindowAnimation
  • 对于WindowAnimation的定义很简单，在style.xml文件中只需要继承Animation Style即可

对于Activity,需要继承Animation Activity Style

当我们从 A1 启动 A2 时，A1 从屏幕上消失，这个动画叫做 android:activityOpenExitAnimation
当我们从 A1 启动 A2 时，A2 出现在屏幕上，这个动画叫做 android:activityOpenEnterAnimation
当我们从 A2 退出回到 A1 时，A2 从屏幕上消失，这个叫做 android:activityCloseExitAnimation
当我们从 A2 退出回到 A1 时，A1 出现在屏幕上，这个叫做 android:activityCloseEnterAnimation

ActivityAnimation稍微复杂一点，但这种复杂带来的转场效果也更好。对于比较追求界面美感的动画，使用ActivityAnimation要好得多，但是一般开发使用windowAnimation即可

  • 在开发中，但Activity只能使用在主题中;窗体动画不仅可以定义在主题中,也可以使用 Activity. overridePendingTransition来设置，而这也是更为常用的一种方式。

Activity转场动画

Activity设置转场动画需要在startActivity和finish之后，意为给开启新视图或关闭旧视图添加转场动画。
Acitivity通过overridePendingTransition(int enterAnim, int exitAnim)实现的，
第一个参数enterAnim，是新的Activity的进入动画的resource ID
第二个参数exitAnim，是旧的Activity(当前的Activity)离开动画的resource ID。
如果上面两个参数没有动画要设置，则用0作为参数。

Fragment转场动画

Fragment的设置需要在transaction.add 或transaction.remove之前。
Fragment有两种方式，一种android提供了默认方法，一种自定义动画

1. Fragment指定标准的转场动画，通过setTransition(int transit)方法

还可以传入的参数
该方法可传入的三个参数是：
TRANSIT_NONE,
TRANSIT_FRAGMENT_OPEN,
TRANSIT_FRAGMENT_CLOSE
分别对应无动画、打开形式的动画和关闭形式的动画。
标准动画设置好后，在Fragment添加和移除的时候都会有。

2. 自定义转场动画
setCustomAnimations()方法，因为Fragment添加时可以指定加入到Back Stack中，所以转场动画有添加、移除、从Back stack中pop出来，还有进入四种情况。
setCustomAnimations()方法必须在add、remove、replace调用之前被设置，否则不起作用。

  • 不使用v4包的情况api>=11,对应属性动画（属性动画资源放在res\animator\目录下）
    根标签是<set>, <objectAnimator>, or <valueAnimator>三者之一。这一点可以从fragment的这个方法看出来onCreateAnimator(int transit, boolean enter, int nextAnim)`返回值是Animator。自定义转场动画时，四个参数的形式setCustomAnimations (int enter, int exit, int popEnter, int popExit)是api13才有的，11只引入了两个动画的形式，即无法指定Back Stack栈操作时的转场动画。
  • 使用v4包，fragment不再局限于api 11之上，低级api也可以使用，这时候专场动画是view动画（res\anim）
    这时FragmentTransaction中的setCustomAnimations()方法，两参数类型和四参数类型都可用，所以一般还是使用v4包的这个版本，一是兼容性比较好，另外View Animation其实基本可以满足转场动画的需要。
    View Animation包含了基本的动画类型。
    根节点可以是：<alpha>, <scale>, <translate>, <rotate>, interpolator element, 或者是<set>。
    用set标签的意思是可以集合多个动画一起执行，也可以自行选择单个动画，如alpha等。
